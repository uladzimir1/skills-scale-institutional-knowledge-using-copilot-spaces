# OctoAcme — Change Control Log

## Purpose
Track all proposed changes to project scope, schedule, process, or deliverables. The Change Control Coordinator owns this log and ensures that all changes follow the defined evaluation and approval process before implementation.

## Change Control Process

1. **Submit** — Anyone on the team may submit a Change Request (CR) using the template below.
2. **Review** — The Change Control Coordinator triages the request and routes it to relevant stakeholders for impact assessment (scope, schedule, cost, quality, risk).
3. **Approve / Reject** — The Project Manager and Product Manager jointly approve or reject the change. For high-impact changes, sponsor sign-off may be required.
4. **Communicate** — The Change Control Coordinator notifies all impacted parties of the decision.
5. **Implement** — Approved changes are added to the backlog and tracked through the standard delivery workflow.
6. **Verify** — The Change Control Coordinator confirms implementation is complete and updates the log.

## Change Request Template

```
Change Request ID: CR-XXX
Date Submitted:    YYYY-MM-DD
Submitted By:      [Name / Role]
Priority:          High / Medium / Low

Description of Change:
[What is being proposed? What will change?]

Reason / Justification:
[Why is this change needed? What problem does it solve or opportunity does it capture?]

Scope Impact:
[What deliverables or work items are affected?]

Schedule Impact:
[Will this affect the timeline or milestones? By how much?]

Cost / Resource Impact:
[Are additional resources, budget, or effort required?]

Quality / Risk Impact:
[Does this introduce new risks or affect existing quality standards?]

Proposed Implementation Plan:
[High-level steps or notes on how the change will be delivered.]

Decision:          Approved / Rejected / Deferred
Decision Date:     YYYY-MM-DD
Decision By:       [Name(s)]
Decision Notes:    [Rationale, conditions, or dependencies]
```

---

## Status Values
- **Pending** — Under review; awaiting decision.
- **Approved** — Approved for implementation; added to backlog.
- **Rejected** — Not approved; reason documented.
- **Deferred** — Decision postponed to a future phase or sprint.
- **Implemented** — Change has been delivered and verified.
- **Cancelled** — Withdrawn before a decision was made.

---

## Escalation
If a change request is not resolved within **5 business days**, the Change Control Coordinator escalates to the Project Manager. For changes with Critical or High business impact, immediate escalation to the sponsor is expected.
